# PR 작성/갱신 규칙

이 문서는 이 저장소의 PR 기록을 일관되게 남기기 위한 운영 규칙입니다.

## 1) PR 본문 작성 언어

- PR 본문은 한국어로 작성합니다.
- 설명은 짧고 명확하게 작성합니다.

## 2) PR 생성 시 필수 항목

PR 생성 시 아래 항목을 반드시 채웁니다.

- 요약: 작업 목적과 결과를 2~4줄로 작성
- 작업 내용: 실제 변경 사항을 항목별로 작성
- 검증: 실행한 검증 방법과 결과를 작성

## 3) 같은 PR에 커밋을 추가한 경우

PR 생성 이후 커밋을 추가하면, 코드를 푸시한 뒤 PR 본문도 반드시 갱신합니다.

- `추가 커밋 반영 내역` 섹션 업데이트
- 새 커밋으로 변경된 작업 내용/검증 결과 반영
- 기존 요약과 작업 내용을 최신 상태로 정리

예시:

```bash
gh pr edit <PR_NUMBER> --body-file .github/pull_request_template.md
```

## 4) 작업 완료 기준

다음 조건을 모두 만족해야 작업 완료로 간주합니다.

- 코드 변경 사항 푸시 완료
- PR 본문이 최신 커밋 기준으로 갱신 완료

## 5) 템플릿

- PR 템플릿: `.github/pull_request_template.md`
- 모든 PR은 템플릿 구조를 따릅니다.
